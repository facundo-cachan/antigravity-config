# STYLE_GUIDE.md

1. Nomenclatura:
   - Métodos: camelCase (ej. convertToJs).
   - Interfaces: PascalCase con prefijo 'I' o sufijo 'Props' (ej. IStyleConverter).

2. Tipado en Documentación:
   - Siempre especificar el tipo de dato entre llaves: {string}, {number}, {Record<string, any>}.

3. Formato de Ejemplos:
   - Deben ser realistas. Si es React Native, usar componentes como View o Text si aplica.
